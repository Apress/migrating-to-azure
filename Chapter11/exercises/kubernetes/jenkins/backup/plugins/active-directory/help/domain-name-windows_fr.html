Option permettant de choisir le domaine Active Directory utilisé pour l'authentification.

Par défaut, Jenkins utilise ADSI pour la communication avec Active Directory, et pour cela la configuration du nom de domaine est optionnelle.

Cependant, et selon des rapports qui n'ont pas été confirmés, ce mécanisme ne fonctionne pas quand l'utilisateur qui exécute Jenkins ne fait pas parti du domaine. (Ce qui peut-être le cas si Jenkins est installé en tant que service par exemple). Dans ce cas veuillez spécifier le nom du domaine ici, et Jenkins utilisera le protocole LDAP standard pour la communication avec Active Directory.
